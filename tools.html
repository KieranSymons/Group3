Tools

Tools used in assessment two are still being used in assessment three with also some new tools the team are using to design the project idea interface and game

  • GitHub group – within GitHub commit trails once again does not show everyone contributions, only the allocated members putting together the website. It lacks in showing the brainstorming process of how the website will look now with all the new assessment section needed to be added in.
  • Visual Studio Code – This tool was used to design and code the IT MOB website, the features that are useful with this tool is being able to code with multiple sheets in one system and adding styling sheets for CSS.
  • Microsoft teams – group chat discussions between members are not shown, here the team members liaising with one another updating each other on work, asking question and receiving support/feedback on work using chat.
  • Figma – within Figma a brainstorming page was created for the allocated members to start communicating about how the design process was going to look and sections that needed to be included within the app for educational purposes. The tool was also used to design and create the interface with a drag and drop system, allowing access to plugins for picture and icon to create a natural theme for the app. Brainstorming for the app reflects in Microsoft video meetings. Figma also allows you to upload your prototypes to community so people outside of your team can view your prototype.
  • BuildBox 3 - is a free game development software system, used to make the recycling game. It is available to download for free on the internet and compatible with both windows and apple.
  • Bravo Studio – Bravo Studio is a tool which uploads Figma files to convert it into an app and then can be uploaded onto IOS/Google play for users to download. An application process is needed here, due to not having the whole app complete this was unable to be done.
  • Bravo Vision – Allows the creator to bring the Figma creation up onto a mobile device to test and trial in app formation, this will show features that are working and not working allow them to adjust where is needed to get the app working correctly.
  • Pxcode – Pxcode was use to convert the Figma prototype into HTML in order to test how the prototype would view within GitHub.

As was the case in our previous assignment, the commit trail of the IT Mob repository, does not reflect the overall contributions of the group. The task of updating and compiling the website was assigned and content was created by all in the team.
